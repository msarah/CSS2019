Media Queries

Media Queries allow your web page to be responsive to different screen sizes. They are defined by setting breakpoints at whatever viewport width your layout should change.

Media Queries have two main components:

media type (optional):

  • all
  • screen
  • print
  • speech

expressions (can have 0 or more expressions):

  • min-width (from this minimum point and upwards)
  • max-width (from this maximum point and downwards)
  • and more ..

examples:

@media [media type] (expression(s)) (embedded in code)

OR

in HTML link to stylesheet **preferred method!

‹ link rel=:stylesheet" href"mq-filename.css" media="(expression)" ›

Why?

  • follows separation of concerns (more modular code!)
  • only delivers the content to the user when they need it
  • with http2, making multip;e requests to a server no longer has a negative impact on performance

D.R.Y - Do Not Repeat Yourself

At 600px OR MORE the text on this page will turn red. (media query using min-width:600px - from this min width and UP)

At 1000px OR LESS the text will turn blue. (media query using max-width: 1000px - from this max width and BELOW)

the order in which you place the links to css media query pages effects the order in which they are applied
